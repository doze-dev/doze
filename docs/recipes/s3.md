# Recipes — S3 (local object storage)

doze embeds a pure-Go, S3-compatible server (gofakes3) — no Docker, no JVM, no
LocalStack. It boots on first request and persists to disk.

## Buckets

```hcl
s3 "media" {
  bucket "uploads" {}
  bucket "thumbs" {
    versioning = true        # honored by clients; the bolt backend is dev-grade
  }
}
```

```sh
# doze run / doze env inject AWS_ENDPOINT_URL_S3 + dummy creds + region.
doze run -- sh -c 'aws s3 ls'                         # lists declared buckets
doze run -- sh -c 'aws s3 cp ./logo.png s3://uploads/'
```

## With the AWS CLI directly

```sh
eval "$(doze env)"                                    # exports AWS_ENDPOINT_URL_S3, AWS_* creds
aws s3 ls
aws s3 cp big.iso s3://uploads/                       # multipart works
aws s3 presign s3://uploads/logo.png                  # presigned GET works
```

## Wiring an app (AWS SDK)

Set a custom endpoint and **path-style addressing** (required for localhost, same
as MinIO/LocalStack). The env doze injects is enough for credentials/region:

```sh
doze run -- node app.js
```

AWS SDK v2 (Go) example:
```go
s3.NewFromConfig(cfg, func(o *s3.Options) {
    o.BaseEndpoint = aws.String(os.Getenv("AWS_ENDPOINT_URL_S3"))
    o.UsePathStyle = true
})
```

boto3:
```python
boto3.client("s3", endpoint_url=os.environ["AWS_ENDPOINT_URL_S3"],
             config=Config(s3={"addressing_style": "path"}))
```

## Notes & limits

- Buckets declared in config are created on boot (`doze up` re-converges them).
  Clients can also `CreateBucket` at will.
- Dev-grade: object CRUD, listing, multipart, and presigned URLs work; bucket
  versioning/IAM are limited. Not a production object store.
